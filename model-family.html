Overview

What You Will Learn

Not all Claude models are the same. Anthropic offers a family of models — Opus, Sonnet, and Haiku — each optimised for a different balance of capability, speed, and cost. There are also multiple ways to access Claude: via chat, API, Claude Code, Claude Desktop (Cowork), and Claude in Chrome. This lesson teaches you how to think about the right model for the right task, and how the different Claudes work together in multi-model systems.

Module 1

The Claude Model Tiers

The Three Tiers at a Glance

Anthropic's current model family has three tiers. Each tier is a deliberate trade-off between intelligence, speed, and cost:

ModelTierBest Described AsSpeedCost
Claude Opus 4.6FlagshipThe most capable — deep reasoning, nuanced judgment, complex tasksSlowestHighest
Claude Sonnet 4.6BalancedThe sweet spot — strong capability with much faster responsesFastMedium
Claude Haiku 4.5LightweightThe quickest and cheapest — great for simple, high-volume, or latency-sensitive tasksFastestLowest
🎯 The Default in Dispatch

Claude Desktop and Dispatch use Sonnet by default — it hits the right balance for most task delegation work. You can override this when you need Opus for harder problems or Haiku for simple high-speed tasks.

Claude Opus — When to Use It

Opus is the flagship model. Use it when the task requires the deepest reasoning, the most nuanced judgment, or the highest accuracy — and you are willing to wait a little longer.

Use Opus When...Example
The task requires sustained multi-step reasoningComplex architecture planning, strategic analysis
Accuracy matters more than speedSecurity audits, legal document review, financial modelling
The problem is ambiguous or requires creativityWriting long-form content, designing novel solutions
You are building a skill that will run many timesGet it right once with Opus, then run it with Sonnet
You are debugging a failing agentOpus is better at diagnosing subtle failure modes

Claude Sonnet — When to Use It

Sonnet is the workhorse. It handles the vast majority of real-world tasks with strong performance and responses that arrive much faster than Opus.

Use Sonnet When...Example
Day-to-day task delegationResearch, file work, document creation, code tasks
Interactive back-and-forthConversations where you want quick responses
Scheduled automationsMorning digests, weekly reports, recurring research
Code generation and reviewBug fixes, feature additions, test writing
Most multi-agent workflowsSubagents doing specific well-defined subtasks

Claude Haiku — When to Use It

Haiku is the fastest and most cost-efficient model. Use it for tasks that are simple, repetitive, or where latency is the primary concern.

Use Haiku When...Example
Simple classification or extractionCategorising a list, extracting a single field from text
High-volume processingProcessing thousands of records, batch file renaming
Real-time or low-latency responsesLive chat, quick lookups, streaming summaries
Routing or triage steps in a pipelineDeciding which agent to hand a task to next
Cost is a primary concernLarge-scale automations where Sonnet would be expensive at scale

Decision Framework

When you are unsure which model to use, work through these questions:

  1. Is the task simple and well-defined? → Haiku
  2. Is the task typical day-to-day work? → Sonnet (default)
  3. Does it require deep reasoning, high accuracy, or complex judgment? → Opus
  4. Will this run thousands of times? → Start with Sonnet, profile cost, drop to Haiku if quality holds
  5. Are you building something mission-critical? → Design with Opus, test whether Sonnet matches quality

Claude Desktop / Cowork — Your Primary Tool

Claude Desktop is the product you are using right now. It layers several capabilities on top of the base Claude model:

  • Dispatch — the orchestrator that coordinates task sessions on your behalf
  • File system access — read and write files on your computer with your permission
  • MCP connectors — integrate with GitHub, Slack, Asana, Google Drive, and more
  • Skills — reusable agent definitions for domain-specific work
  • Scheduled tasks — recurring automations that run without you
  • Code sessions — full Claude Code sessions with git worktree isolation
💡 Key Insight

Claude Desktop is not just a chat interface — it is an orchestration platform. The model (Sonnet by default) is the reasoning engine, but Dispatch, MCP, and skills are what make it a genuine productivity system.

Claude in Chrome

Claude in Chrome is a browser automation layer. When connected, it gives Dispatch the ability to take screenshots, click and fill forms on any website, extract structured data from web pages, and automate multi-step browser workflows.

🔌 Setup Reminder

Claude in Chrome requires the Chrome extension to be installed and signed in. Once connected, Dispatch detects it automatically and can use browser tools as part of any task.

Module 3

How Models Work Together

Multi-Model Workflows

Sophisticated agent systems often use different models at different stages — a technique called model routing. The idea is to match the intelligence level of the model to the complexity of the subtask.

StageRecommended ModelReason
Triage / routingHaikuSimple decision: which agent should handle this?
Data extraction / parsingHaikuWell-defined task, high volume, low complexity
Research and synthesisSonnetModerate complexity, needs real reasoning
Code generation and reviewSonnetStrong but does not need Opus for typical tasks
Architecture / planningOpusHigh complexity, nuanced judgment required
Final verification / QAOpusCatch what Sonnet might have missed

The Orchestrator-Worker Pattern

The most common multi-model pattern uses a capable model to orchestrate and a lighter model for routine work:

  1. Sonnet (orchestrator) receives the high-level goal and breaks it into subtasks
  2. Haiku (workers) handle simple, well-defined subtasks in parallel
  3. Sonnet collects and synthesises the results
  4. Opus (reviewer) does a final quality check on high-stakes outputs
💰 Cost Efficiency

This pattern can cut costs by 60–80% on high-volume workflows compared to running everything through Sonnet or Opus, while maintaining near-identical output quality for most tasks.

How Dispatch Handles Model Selection

  • Default sessions use Sonnet unless you specify otherwise
  • Request Opus explicitly: 'Use Opus for this — it needs deep reasoning'
  • Request Haiku explicitly: 'Use the fastest model — this is a simple extraction'
  • Code sessions started via Dispatch inherit the model you specify, defaulting to Sonnet
🔄 How to Override the Model

In Dispatch, just say which model you want. The model strings are:
claude-opus-4-6  |  claude-sonnet-4-6  |  claude-haiku-4-5-20251001

⚡ Default Rule

When in doubt, use Sonnet. It is fast, capable, and handles the vast majority of tasks excellently. Only reach for Opus when you genuinely need it, and Haiku when speed or cost is the primary driver.

Practice

Exercises

Exercise 1

Model Audit (10 min)

Review the automations and scheduled tasks you have set up. For each one, ask: is Sonnet the right model, or should this use Haiku (simpler task) or Opus (higher stakes)? Note any you would change.

Exercise 2

Opus vs. Sonnet Comparison (15 min)

Take a complex research or analysis task. Run it once with Sonnet and once with Opus ('use Claude Opus for this'). Compare the outputs: depth, accuracy, nuance. Is Opus meaningfully better for this type of task?

Exercise 3

Haiku for Batch Work (10 min)

Find a simple, repetitive task you do often. Run it with Haiku ('use the fastest model'). Verify the quality is acceptable for your use case.

Exercise 4

Design a Multi-Model Workflow (20 min)

Take any complex automation you have built or want to build. Redesign it as a multi-model pipeline: which steps use Haiku, which use Sonnet, which (if any) use Opus? Write out the pipeline as a Dispatch prompt.

Key Takeaways

  • Opus: the most capable, slowest, and most expensive — for genuinely hard problems.
  • Sonnet: the default and the right choice for most tasks — fast, capable, balanced.
  • Haiku: fastest and cheapest — for simple, high-volume, or latency-critical tasks.
  • Claude Desktop is an orchestration platform, not just a chat interface.
  • Multi-model workflows route tasks to the right model at each stage — maximising quality and minimising cost.
  • When in doubt: start with Sonnet, add Opus where accuracy demands it, and Haiku where speed or volume demands it.
