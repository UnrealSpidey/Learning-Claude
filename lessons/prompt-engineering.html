Overview

What You Will Learn

Prompt engineering is the art of communicating with Claude in ways that produce reliable, accurate, and useful results. This lesson covers the most effective techniques — from basic structuring to advanced chain-of-thought methods — so you can get the output you actually want, every time.

Module

Why Prompt Engineering Matters

How Claude Processes Your Input

Claude reads your entire message and generates a response that is the most likely continuation given everything it has seen. Every word you include — and every word you omit — shapes the output.

Input QualityTypical Output
Vague goal, no contextGeneric, shallow, often off-target
Clear goal, some contextRelevant but may miss nuance
Clear goal + context + examplesSpecific, accurate, formatted correctly
Clear goal + context + examples + constraintsConsistent, reliable, production-ready

The Five Elements of a Strong Prompt

  1. Role — who Claude should act as (e.g. senior Python developer, technical writer)
  2. Goal — the specific outcome you want
  3. Context — background information Claude needs to make good decisions
  4. Format — how the output should be structured
  5. Constraints — what Claude should NOT do
⚖️ Weak vs. Strong Example

WEAK: 'Summarize this code.'

STRONG: 'You are a senior software engineer reviewing a Python codebase for a new team member. Summarize the attached file in plain English, covering: (1) what it does, (2) key functions and their purpose, (3) any risks or code smells. Use bullet points. Do not reproduce the code.'

Module

Core Techniques

Chain-of-Thought Prompting

Asking Claude to reason step-by-step before giving a final answer dramatically improves accuracy on complex tasks.

  • Add: 'Think through this step by step before answering'
  • Add: 'Show your reasoning, then give your final answer'
  • Add: 'List any assumptions you are making'
🔗 When to Use It

Most valuable for: multi-step problems, tasks with ambiguous inputs, calculations, debugging, and any time accuracy matters more than speed.

Few-Shot Examples

Providing 2–3 examples of input/output pairs is often more effective than any amount of description.

ApproachEffectiveness
Description only ('be concise')Moderate — Claude interprets 'concise' differently each time
One exampleGood — Claude infers the pattern
2–3 varied examplesExcellent — Claude generalises reliably
Examples + negative exampleBest — Claude knows what to avoid too

Negative Examples & Constraints

Telling Claude what NOT to do is as powerful as telling it what to do:

  • 'Do not use bullet points' — forces prose
  • 'Do not invent details that are not in the source' — prevents hallucination
  • 'Do not explain what you are doing, just do it' — removes unnecessary preamble
  • 'Do not summarise the problem back to me before answering' — saves time

XML Tags for Structured Output

Asking Claude to wrap its response in XML tags makes outputs easy to parse, especially when building automated workflows.

📄 Example

Prompt: 'Analyse this code and respond in this format:
<summary>One sentence summary</summary>
<risks>List of risks</risks>
<recommendations>List of recommendations</recommendations>'

Advanced Techniques

Prompt Chaining

Break complex tasks into a chain of simpler prompts, where each prompt builds on the previous output:

  1. Prompt 1: Extract the raw data or information
  2. Prompt 2: Analyse or transform it
  3. Prompt 3: Format the final output

Self-Consistency

For high-stakes outputs, ask Claude to generate multiple independent responses and then synthesise the best answer.

  • 'Generate three different approaches to this problem, then recommend the best one with justification'
  • 'Answer this question twice independently, then compare your two answers and resolve any discrepancies'

Asking Claude to Critique Its Own Output

  • 'Now review your answer for accuracy and completeness. What is missing or wrong?'
  • 'Rate your response 1–10 for accuracy. What would make it a 10?'
  • 'What assumptions did you make? Are any of them wrong?'

Common Mistakes & Fixes

Prompt Anti-Patterns

Anti-PatternWhat Goes WrongFix
Asking multiple questions at onceClaude prioritises some, skips othersOne question per prompt; chain if needed
Vague quality bar ('be good')Claude has no measurable targetGive a concrete example of good output
No output format specifiedOutput format varies every timeAlways specify format explicitly
Too much irrelevant contextClaude focuses on the wrong informationTrim to only what is necessary

Iterating on Prompts

The Prompt Iteration Cycle

  1. Write a first-draft prompt
  2. Run it and identify the first point of failure
  3. Hypothesise why it failed: missing context? Wrong format? Vague goal?
  4. Change one variable at a time
  5. Re-run and compare
  6. Repeat until output meets your standard
📏 Rule of Thumb

If you need to change the prompt more than 5 times for the same task, the task is too complex for a single prompt. Break it into a chain.

Practice

Exercises

Exercise 1

Exercise 1 — Rewrite a Weak Prompt (10 min)

Take a prompt you have used before that gave a mediocre result. Apply the five-element structure (role, goal, context, format, constraints) and re-run it. Compare the outputs.

Exercise 2

Exercise 2 — Add Few-Shot Examples (10 min)

Pick any task you do repeatedly with Claude. Write 2–3 input/output examples and add them to your prompt. Note the improvement in consistency.

Exercise 3

Exercise 3 — Chain of Thought (10 min)

Ask Claude a complex question without chain-of-thought. Then add 'Think step by step before answering.' Compare the depth and accuracy of both responses.

Exercise 4

Exercise 4 — Self-Critique Loop (15 min)

Get Claude to produce any output. Then prompt: 'Review your answer for accuracy and completeness. What would you change?' Apply the feedback and compare the results.

Key Takeaways

  • Strong prompts have five elements: role, goal, context, format, constraints.
  • Examples outperform descriptions every time.
  • Chain-of-thought improves accuracy on hard tasks.
  • XML tags make outputs parseable for automated workflows.
  • Iterate one variable at a time when debugging prompts.
  • Save your best prompts as skills.
