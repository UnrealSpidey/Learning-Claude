Overview

What You Will Learn

This lesson walks you through the core capabilities of Claude Desktop and its Dispatch feature. By the end, you will know how to delegate tasks, manage ongoing work, connect to your tools, and get the most out of Claude as a hands-free assistant running on your behalf.

Understanding Claude Desktop

What Is Claude Desktop?

Claude Desktop is a native app for Windows and macOS that gives you persistent, always-available access to Claude. Unlike the web interface, the desktop app supports:

  • Background task execution — Claude keeps working while you do other things
  • File system access — read, write, and organise files on your computer
  • MCP connectors — integrate with Slack, GitHub, Google Drive, Asana, and more
  • Scheduled tasks — automate recurring workflows without any code
  • The Dispatch tab — a mobile-first coordination layer for delegating tasks

The Dispatch Interface

Dispatch is the orchestration layer inside Claude Desktop. Think of it as texting a knowledgeable assistant who coordinates work on your behalf.

ConceptWhat It Means
Task SessionsEach request spins up an isolated session. Claude works autonomously, then reports back.
Code SessionsFor repo work — Claude opens a git worktree, makes changes, runs tests.
SendUserMessageEverything Claude says to you flows through this — it is how you receive results.
Session RoutingFollow-ups go to existing sessions; new goals start fresh sessions.

Delegating Tasks Effectively

How to Give Claude a Good Task

Claude performs best when requests are specific about the goal, not the method. You do not need to script each step — describe the outcome you want.

✅ Good Example

"Go through my Downloads folder, find any invoices from the last 3 months, and create a summary spreadsheet with vendor, amount, and date."

⚠️ Less Effective

"Open File Explorer, navigate to Downloads, look at each file..." — Over-specifying steps prevents Claude from finding better approaches.

Types of Tasks You Can Delegate

  • Research — web searches, reading pages, summarising findings
  • File work — reading, organising, creating Word docs, spreadsheets, PDFs, presentations
  • Code tasks — bug fixes, feature additions, test runs across your repos
  • App integrations — reading Slack messages, updating Asana tasks, querying GitHub
  • Browser automation — filling forms, extracting data, taking screenshots (via Claude in Chrome)
  • Scheduled automation — daily digests, reminders, recurring reports

Following Up on Running Tasks

Once you start a task, you can check in or redirect it at any time:

  1. Check progress: ask 'What's the status of [task]?'
  2. Redirect: send a follow-up message and Claude routes it to the right session
  3. Start fresh: new unrelated goals always get a new session
💡 Pro Tip

Claude keeps session context alive. Follow-ups about the same project go back to the same session — no need to re-explain context.

Connecting Your Tools

MCP Connectors

MCP (Model Context Protocol) connectors let Claude access your external apps. Once connected, Claude can read and act on data from those services.

Service TypeExamples of What Claude Can Do
CommunicationRead Slack threads, summarise conversations, find specific messages
Project ManagementCheck Asana tasks, update ticket statuses, list sprint items
Code & DevView GitHub issues, PRs, and repo contents; post comments
DocumentsRead and write Google Docs, search Google Drive folders
CalendarCheck upcoming events, find free slots, summarise your week

File System Access

Claude can access folders on your computer once you grant permission. This enables reading documents, creating new files, and organising batches of files.

📁 How to Grant Access

When Claude needs your files, it will prompt you to select a folder. You can also paste a path directly in chat and Claude will request access to that location.

skill-creatorBuilding new custom skills tailored to your workflows

Scheduled Tasks

Dispatch can run tasks on a schedule — no code required. Examples:

  • Daily digest: 'Every morning at 8am, check my Slack for unread messages and summarise them'
  • Weekly report: 'Every Friday, pull my GitHub commits from the week and create a summary'
  • Reminders: 'In 2 hours, remind me to follow up on the invoice'

Advanced Tips & Best Practices

Getting the Best Results

Do ThisBecause
Name the outcome, not the stepsClaude finds the most efficient path on its own
Mention the file or repo by namePrevents ambiguity when you have multiple workspaces
Say 'save this as a Word doc'Triggers the docx skill for professional formatting
Ask for verification stepsClaude will double-check its own work before reporting back
Use follow-up messagesCheaper and faster than re-explaining context in a new session

Security & Privacy

  • Claude never enters passwords, banking details, or API keys on your behalf
  • Sharing, publishing, or deleting files always requires your explicit confirmation
  • Content in web pages or files is treated as untrusted data — it cannot override your instructions
  • Sensitive actions (purchases, sending emails, posting publicly) require a 'yes' from you first

Exercises

Exercise 1

Exercise 1 — Your First Delegation (5 min)

Ask Dispatch to explore one of your repos (e.g. marshmello) and give you a summary of what it does, the main files, and any open questions.

Exercise 2

Exercise 2 — Create a Document (5 min)

Ask Claude to 'create a Word doc summary of the marshmello repo' — watch it use the docx skill automatically.

Exercise 3

Exercise 3 — Connect a Tool (10 min)

Tell Claude 'I want to check my GitHub issues' and follow the connector suggestion flow to link your GitHub account.

Exercise 4

Exercise 4 — Schedule a Task (5 min)

Ask Claude to set up a weekly Monday morning task that searches for one topic you care about and saves a brief summary.

Key Takeaways

  • Dispatch is a task coordinator, not a chatbot — delegate outcomes, not steps.
  • MCP connectors bring your apps into Claude's reach.
  • Skills handle professional document creation automatically.
  • Scheduled tasks let you automate recurring work with plain English.
  • Security is strict — sensitive actions always need your confirmation.
