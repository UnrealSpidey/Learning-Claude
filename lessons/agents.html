Overview

What You Will Learn

This lesson teaches you how to design, build, and orchestrate agents using Claude and the Claude Agent SDK. You will learn what makes a great agent, how to compose multi-agent systems, how to write effective prompts and skills, and how to evaluate agent performance.

What Is an Agent?

Agents vs. Chatbots

A chatbot responds to a single message. An agent pursues a goal autonomously, using tools and making decisions across multiple steps.

DimensionAgentChatbot
ScopeMulti-step tasksSingle exchanges
ToolsCalls APIs, reads files, searchesJust generates text
MemoryTracks state across stepsStarts fresh each turn
AutonomyDecides what to do nextWaits for the next prompt

The Agent Loop

Every agent operates on a core loop:

  1. Receive a goal or task
  2. Decide which tool or action to take next
  3. Execute the action (web search, file read, API call, etc.)
  4. Observe the result
  5. Decide: is the goal complete? If not, return to step 2
  6. Report results back to the user or calling system
💡 Why This Matters

Good agent design means giving the agent the right tools, a clear goal, and enough context to make good decisions at each step. Vague goals and missing tools cause agents to loop or hallucinate.

Writing Effective Agent Prompts

Anatomy of a Good Agent Prompt

📋 Prompt Structure Template

1. Role — what kind of agent is this?
2. Goal — what is the agent trying to achieve?
3. Tools — which tools should it use, and in what order?
4. Output format — what should the final result look like?
5. Constraints — what should the agent NOT do?

High-Performance Prompt Principles

  • Be specific about the goal, not the steps — let the agent decide the path
  • Provide examples of good and bad outputs — more powerful than descriptions
  • Include negative examples — tell the agent what to avoid
  • Ask for step-by-step reasoning before the final answer
  • Request output in a specific structure (XML tags, JSON, markdown headers)
  • Give the agent a way to ask for clarification rather than guess

Common Prompt Anti-Patterns

Anti-PatternWhy It Fails
'Do your best'No measurable success criteria — the agent doesn't know when to stop
Over-specifying stepsPrevents the agent from finding a better approach
Missing contextAgent makes assumptions that don't match your environment
No output formatResults are inconsistent and hard to act on
No constraintsAgent may over-reach, make changes you didn't want, or loop

Building Custom Skills

What Is a Skill?

A skill is a reusable agent definition stored as a SKILL.md file. It packages a specific capability so you can invoke it repeatedly without re-prompting.

Your setup includes the skill-creator skill. Use it to build new skills by describing what the skill should do, what inputs it needs, what tools it should use, and what the output should look like.

Anatomy of a SKILL.md

  1. Frontmatter — title, description, trigger conditions
  2. Context — what environment the skill expects
  3. Instructions — step-by-step behaviour for the agent
  4. Output specification — format and contents of the final deliverable
  5. Error handling — what to do when things go wrong

Multi-Agent Orchestration

Orchestration Patterns

PatternHow It Works
SequentialAgent A completes its work, passes output to Agent B, then C. Use when each step depends on the last.
ParallelMultiple agents run simultaneously on independent subtasks. Use to speed up large research or analysis jobs.
SupervisorAn orchestrator agent breaks the goal into subtasks and spawns specialised subagents for each.
VerificationAfter the main agent finishes, a second agent independently checks the output.
IterativeAn agent produces a draft, a second agent critiques it, the first revises. Repeat until quality threshold met.

Evaluation & Iteration

How to Know If Your Agent Is Working

Good agents need measurable success criteria. Before building, define:

  • What does a correct output look like? (example)
  • What does a wrong or incomplete output look like? (counter-example)
  • What is the minimum acceptable quality bar?
📊 Eval Framework

For each skill or agent, create 3–5 test cases: one easy, one medium, one hard, one edge case, and one adversarial. Run all five and look for patterns in where the agent fails.

Common Failure Modes & Fixes

Failure ModeSymptomFix
Under-specified goalAgent asks many questions or produces shallow outputAdd concrete examples of desired output
Missing toolAgent says 'I cannot access...'Add the right tool or MCP connector
Context overflowAgent forgets early instructions mid-taskBreak into smaller sequential agents
HallucinationAgent confidently states wrong factsAdd a verification step; require citations
Practice

Exercises

Exercise 1

Exercise 1 — Locate & Read a Skill (10 min)

Find your custom research skill. Read its SKILL.md and identify: the goal statement, the tools it uses, and the output format. Write one improvement you would make.

Exercise 2

Exercise 2 — Improve a Skill with skill-creator (15 min)

Tell Claude: 'Use the skill-creator to improve my research skill to also produce a structured markdown summary with key findings, open questions, and recommended next steps.'

Exercise 3

Exercise 3 — Build a New Skill from Scratch (20 min)

Ask Claude to build a 'code review' skill that reads a repo, identifies potential bugs, and outputs a prioritised list as a Word document.

Exercise 4

Exercise 4 — Multi-Agent Workflow (20 min)

Chain two agents: first use the Explore agent to map the marshmello repo structure. Then feed those findings to the Plan agent to produce an implementation plan for a new feature.

Exercise 5

Exercise 5 — Evaluate Your Agent (15 min)

Take the code review skill from Exercise 3. Run it on three different repos. Score each output 1–5. Identify the most common failure and update the skill prompt to address it.

Key Takeaways

  • Agents are goal-directed loops, not chatbots.
  • Good prompts specify the goal, tools, output format, and constraints — with examples.
  • Skills are reusable agent definitions: invest in making them specific and testable.
  • Multi-agent systems scale to complex work — use orchestration patterns intentionally.
  • Evaluation is not optional — define success criteria before you build, not after.
