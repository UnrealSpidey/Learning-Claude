Overview

What You Will Learn

This lesson teaches you how to design, connect, and maintain a personal automation stack using Claude Desktop and Dispatch. By the end, you will have a set of recurring tasks and connected services that keep you informed and productive — without daily manual effort.

What Is a Personal Automation Stack?

Definition

A personal automation stack is a set of connected tools, recurring tasks, and agent workflows that run on your behalf — reading your data, synthesising it, and delivering actionable insights — without you having to ask each time.

Without AutomationWith Automation Stack
Check GitHub manually every morningClaude summarises open PRs and new issues at 8am
Dig through Slack to catch upClaude delivers a digest of important threads overnight
Manually track project tasksClaude checks Asana and flags overdue or blocked items
Create status reports manuallyClaude compiles reports from your tools on a schedule

The Three Layers of Your Stack

  1. Data sources — where your information lives (GitHub, Slack, Asana, files, web)
  2. Agents — Claude tasks that read, process, and act on that data
  3. Outputs — summaries, documents, alerts, or actions delivered to you
🏗️ Design Principle

Every automation should answer one question you would otherwise have to answer manually. Start with the questions you ask most often.

Designing Recurring Tasks

Anatomy of a Good Automated Task

Before scheduling anything, define these four things:

  1. Trigger: when does it run? (daily 8am, every Monday, on demand)
  2. Data source: what does it read? (GitHub, Slack, files, web)
  3. Processing: what does it do with the data? (summarise, filter, analyse)
  4. Output: what does it produce? (Word doc, markdown file, notification)

Chaining Workflows

Sequential Chains

The output of one task becomes the input of the next:

  1. Task 1: Fetch raw data from a source
  2. Task 2: Analyse or filter that data
  3. Task 3: Format and save the final output

Parallel Chains

Multiple tasks run at the same time and their outputs are merged. Use this when gathering data from multiple independent sources:

  • Fetch GitHub issues AND Slack messages AND calendar events simultaneously
  • Merge all three into a single morning briefing
⚡ Efficiency Tip

Parallel tasks complete faster than sequential ones. When your data sources are independent, always run them in parallel.

Building Your Morning Digest

Sample Digest Prompt

📋 Template Prompt

Every day at [TIME], check: (1) GitHub — list any new issues or PRs in my repos. (2) Slack — summarise any messages mentioning my name or marked urgent. (3) My calendar — list today's events. Combine into a single morning briefing. Format: plain text, under 300 words. Flag anything that needs action today.

Maintaining Your Stack

Stack Hygiene

Review your stack monthly:

  • Which tasks are still valuable? Which are producing noise?
  • Are any outputs being ignored? If so, the task is probably wrong.
  • Have your connected services changed? (New repos, new Slack channels)
  • Are there new questions you ask manually that could be automated?
Practice

Exercises

Exercise 1

Exercise 1 — Map Your Data Sources (10 min)

List every tool or service you check manually on a daily or weekly basis. For each, write: what question you are answering, and how often it actually changes. These are your automation candidates.

Exercise 2

Exercise 2 — Connect One MCP Service (15 min)

Pick the tool you check most often. Ask Dispatch to connect it. Walk through the connector installation. Run a test query to confirm it works.

Exercise 3

Exercise 3 — Build Your Morning Digest (20 min)

Using the template from Module 5, create your morning digest task. Schedule it for tomorrow morning. After it runs, review the output and adjust the prompt if needed.

Exercise 4

Exercise 4 — Chain Two Tasks (20 min)

Pick any two tasks from the template list. Design a chain: what does Task 1 produce that Task 2 needs? Create both in Dispatch and test the chain manually before scheduling.

Key Takeaways

  • A personal automation stack has three layers: data sources, agents, and outputs.
  • MCP connectors unlock your external tools.
  • Every automation should answer one question you currently answer manually.
  • Start with the morning digest — it is the highest ROI automation.
  • Review and prune your stack monthly.
